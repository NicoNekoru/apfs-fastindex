\chapter*{Preface}
\addcontentsline{toc}{chapter}{Preface}

This manual records what the \texttt{apfs-fastindex} project has
learned about the Apple File System on disk, and about the problem
of enumerating a single APFS volume quickly and correctly. It is
written for a careful engineer who wants to read APFS metadata
directly. Familiarity with copy-on-write filesystems and B-trees is
assumed; APFS-specific knowledge is not.

\section*{Claim Labels}

Three labels appear throughout the text and are never blurred.

\begin{description}
  \item[Spec.] Backed by Apple's public reference\footnote{\emph{Apple
    File System Reference}, retrieved 2026-05-13,
    \url{https://developer.apple.com/support/downloads/Apple-File-System-Reference.pdf}.}
    or by another durable specification.
  \item[Observation.] Backed by a controlled experiment in this
    project, or by behaviour that converges across multiple
    independent open-source readers
    (\texttt{linux-apfs-rw}, \texttt{apfs-fuse}, \texttt{dissect.apfs},
    \texttt{go-apfs}, \texttt{libfsapfs}, the Sleuth Kit's APFS
    module).
  \item[Hypothesis.] Plausible, useful for design, not yet proven.
    A hypothesis does not justify product behaviour; the parser does
    not emit a row that depends only on a hypothesis-class claim.
\end{description}

Where Spec and Observation disagree, Observation wins. The reference
describes intent; the running system writes what is on disk, and the
two occasionally disagree.

\section*{Oracles}

Each feature owns its own oracle.

\begin{description}
  \item[Namespace.] POSIX traversal of the same volume at the same
    selected state.
  \item[Logical size.] \texttt{st\_size} per inode; for symlinks,
    the target byte length returned by \texttt{readlink}.
  \item[Allocated size.] \texttt{st\_blocks * 512}.
  \item[FS-record body.] Field-equality between independent
    decoders against the same raw container.
  \item[Real (clone-deduplicated) size.] $\sum
    \texttt{phys\_ext.length} \cdot \texttt{block\_size}$ from
    the on-disk extent-reference tree (no public per-file macOS
    oracle exists; \texttt{du -A} reports apparent size and does
    not dedup clones).
  \item[Incremental correctness.] A fresh full reparse of the
    same selected state.
\end{description}

Every result names the oracle that verified it.

\section*{Fail-Closed Discipline}

Outside the tested allowlist the parser stops with a typed error
and the caller picks a fallback. Plausible-but-unproven is treated
as ``do not emit.'' The cost of a silently wrong row is unbounded:
it propagates into directory aggregates, into the treemap, and into
decisions the user takes from the treemap. The cost of refusing a
row is bounded: the fallback path produces a correct answer through
supported APIs.

\section*{Scope}

Each chapter takes one face of APFS, states what is known, presents
the evidence, and notes what remains open. Where the implementation
makes a non-obvious choice the choice is explained from first
principles. What is not here: a tour of the repository, a list of
files, a chronology of commits. Those live in the repository; the
manual cites them only when citation supports a claim.
